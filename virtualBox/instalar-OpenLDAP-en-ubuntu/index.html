Cómo instalar el servidor OpenLDAP en Ubuntu

En este artículo vamos a realizar la instalación del servidor OpenLDAP, llamado slapd y sus utilidades, que es el servidor DNS para Linux bajo la versión de Debian.

  1. Para comenzar, debemos tener conexión a Internet y una IP fija para nuestro servidor (ya la tenemos configurada). Ejecutamos el siguiente comando:

    $ sudo apt-get install slapd ldap-utils

  2. Al iniciar la instalación nos aparecerá la siguiente imagen, donde se nos pedirá una contraseña para el administrador del directorio LDAP. Pondremos como contraseña admin y la confirmaremos.(En nuestro caso ubuntu2023)

  3. Ejecutando el comando siguiente: $ sudo netstat –natup, podemos comprobar si la instalación e iniciación esta correcta y el puerto del servicio está abierto. El puerto debe ser el 389.

  4. A continuación, para comenzar la configuración manual, ejecutamos el siguiente comando:

    $ sudo dpkg-reconfigure slapd

  5. Al ejecutar el comando anterior, aparecerá una interfaz de usuario azul en la ventana del terminal. Donde seleccionaremos la opción «No» en la ventana de la interfaz de usuario. Si eliges accidentalmente la opción «Sí», vuelve a ejecutar el comando de configuración para solucionarlo.

  6. En la siguiente ventana nos pedirá teclear el dominio DNS para construir el DN base del directorio LDAP. Nosotros teclearemos nuestro dominio micentro.edu.

  7. Seguidamente teclearemos el nombre de la organización, donde pondremos trassierra

  8. De nuevo nos pedirá la contraseña de administrador (en nuestro caso ubuntu2023) dos veces.

  9. Después de elegir un proveedor de base de datos, la interfaz de usuario nos preguntará si deseamos eliminar la base de datos si se utiliza el comando apt remove –purge . Elije «No» para garantizar que sus datos permanezcan seguros.

  10. En la siguiente interfaz de usuario nos informará que todavía existen algunos archivos en el directorio /var/lib/ldap en nuestro sistema y que, dejar estos archivos allí, es peligroso y puede causar problemas. Para resolver el problema, la herramienta de configuración Slapd nos indica que movamos la base de datos anterior a /var/backups, seleccionando la opción «Sí».

  11. Para finalizar, Ubuntu nos indicará que el proceso de configuración ha terminado.
